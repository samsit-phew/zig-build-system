# build.zig.zon

`build.zig.zon` is the package manifest. It declares your package's identity and its dependencies.

```zig
.{
    .name = .myapp,           // package name (atom, not string)
    .version = "0.1.0",
    .minimum_zig_version = "0.16.0",
    .fingerprint = 0xdeadbeef,  // generated by zig, don't touch
    .paths = .{""},             // what to include when publishing ("" = everything)

    .dependencies = .{
        // local package
        .princ = .{
            .path = "../princ",
        },

        // remote package
        .zap = .{
            .url = "https://github.com/zigzap/zap/archive/abc123.tar.gz",
            .hash = "1220...",
        },
    },
}
```

## Getting the hash for a remote package

```bash
# add the url without the hash line, then run:
zig build
# zig will error and tell you the correct hash to paste in
```

Or use the fetch command:

```bash
zig fetch --save https://github.com/zigzap/zap/archive/abc123.tar.gz
# automatically adds the url + hash to your build.zig.zon
```

## Fields

- `.name` — atom (`.myapp`), not a string. Used as the package identifier.
- `.version` — semver string.
- `.minimum_zig_version` — zig will error if the user's version is older.
- `.fingerprint` — generated by `zig init`, don't change it.
- `.paths` — what files to include if someone depends on you. `""` means the whole repo.
